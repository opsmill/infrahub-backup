This guide provides step-by-step instructions for installing the Infrahub Ops CLI binaries.

## Supported environments

- Linux (amd64 and arm64)
- macOS (amd64 and arm64)

## Prerequisites

- Docker (if you are using a Docker Infrahub setup)
- kubectl (if you are using k8s)

## Install & run the CLI

1. Download the executables you need

```bash
curl -L https://infrahub.opsmill.io/ops/$(uname -s)/$(uname -m)/infrahub-backup -o infrahub-backup
curl -L https://infrahub.opsmill.io/ops/$(uname -s)/$(uname -m)/infrahub-environment -o infrahub-environment
curl -L https://infrahub.opsmill.io/ops/$(uname -s)/$(uname -m)/infrahub-taskmanager -o infrahub-taskmanager
curl -L https://infrahub.opsmill.io/ops/$(uname -s)/$(uname -m)/infrahub-version -o infrahub-version
chmod +x infrahub-*
```

2. Run a command

```bash
./infrahub-environment detect
```

### Available binaries

| Binary | Purpose |
| --- | --- |
| `infrahub-backup` | Create and restore Infrahub backups |
| `infrahub-environment` | Inspect and detect deployment environments |
| `infrahub-taskmanager` | Run housekeeping operations against Prefect |
| `infrahub-version` | Print version information |

### Configuration


Set the following environment variables as needed:

| Variable                              | Description                                      | Default                                  |
|---------------------------------------|--------------------------------------------------|------------------------------------------|
| `INFRAHUB_PROJECT`                    | Docker Compose project override                  | Auto-detected                             |
| `INFRAHUB_K8S_NAMESPACE`              | Kubernetes namespace override                    | Auto-detected                             |
| `INFRAHUB_DB_DATABASE`                | Neo4j database name                              | Fetched from running instance OR `neo4j` |
| `INFRAHUB_DB_USERNAME`                | Neo4j username                                   | Fetched from running instance OR `neo4j` |
| `INFRAHUB_DB_PASSWORD`                | Neo4j password                                   | Fetched from running instance OR `admin` |
| `PREFECT_API_DATABASE_CONNECTION_URL` | Prefect/Postgres connection string               | Fetched from running instance            |
| `BACKUP_DIR`                          | Directory for storing backups                    | `./infrahub_backups`                     |
| `INFRAHUB_LOG_FORMAT`                 | Log output format (`text` or `json`)             | `text`                                   |

You can also set the same overrides at runtime using CLI flags:

- `<binary> --project <docker_project>` to target a specific Docker Compose deployment.
- `<binary> --k8s-namespace <namespace>` to target a specific Kubernetes deployment.
